# 📊 Melhorias para o Micro SaaS de Validação de Dicionário de Dados

## ✅ Visão Geral
Este documento consolida todas as melhorias recomendadas para o desenvolvimento do micro SaaS de validação de dicionários de dados.

O objetivo é evoluir o sistema de um MVP funcional para uma plataforma robusta de governança de dados.

---

# 🔥 1. Melhorias Críticas

## 1.1 Tabela de Resumo de Validação
Problema:
A tabela `validations` armazena respostas individuais, mas não permite consolidar o status final de cada campo.

Solução:
Criar uma tabela de agregação.

```sql
field_validation_summary (
    field_id int,
    total_validations int,
    approved_count int,
    rejected_count int,
    status_final varchar, -- approved, rejected, conflict
    score decimal
)
```

Benefícios:
- Permite identificar consenso
- Detecta conflitos
- Base para dashboard confiável

---

## 1.2 Enriquecimento das Validações

Problema:
Validação binária (approved/rejected) é limitada.

Solução:
Expandir para múltiplos critérios:

- used (boolean)
- required (boolean)
- correct_name (boolean)
- correct_origin (boolean)
- has_business_rule (boolean)

Benefícios:
- Diagnóstico mais rico
- Maior aderência ao negócio

---

## 1.3 Score de Qualidade por Campo

Criar cálculo automático:

```text
score =
  20% used +
  20% required +
  20% origin +
  20% consensus +
  20% rule_defined
```

Classificação:

| Score | Classificação |
|------|-------------|
| 90+ | Confiável |
| 60-89 | Atenção |
| < 60 | Crítico |

# 🚀 2. Melhorias Importantes

## 2.1 Versionamento de Dicionários

Adicionar colunas:

```sql
version int
parent_id int
```

Benefícios:
- Histórico de alterações
- Comparação entre versões
- Auditoria

---

## 2.2 Melhoria nas APIs

Novos endpoints recomendados:

```http
GET /api/fields/:id/summary
GET /api/fields/critical
GET /api/dictionaries/:id/export
```

Exportações:
- JSON validado
- CSV
- SQL DDL

---

## 2.3 Experiência do Usuário (UX)

Melhorias:
- Agrupar campos por tabela/categoria
- Filtros por status (pendente/aprovado)
- Busca por campo
- Ordenação por criticidade
